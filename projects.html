Viral Food Trend Forecaster

Python Streamlit Prophet scikit-learn Google Trends Pandas

A full end-to-end data science app that predicts which food trends are emerging before they go mainstream. I track 150+ keywords — from birria tacos to pistachio cream — by pulling 5 years of weekly Google Trends data via the pytrends API, batching requests to work around rate limits.

The ML pipeline has two layers. A Random Forest Regressor (200 estimators) is trained on engineered features — growth rate, acceleration, momentum, volatility, and ratio to historical peak — to predict each keyword's Google Trends score 8 weeks ahead. Data-driven thresholds (computed from training quantiles, not hardcoded rules) then classify each trend into a phase: emerging, peaking, declining, stable, or dormant. A separate Facebook Prophet model generates a 12-week forecast with an 80% confidence interval for any selected keyword.

The Streamlit dashboard lets users search any food in real time, see its current phase and 4-week growth, and view a live forecast chart — all cached so the app stays fast between interactions.

Contextual Embeddings Explorer

Python BERT PyTorch NLP scikit-learn

I built a pipeline to explore how BERT's contextual embeddings capture word meaning differently depending on context. I chose the word "layer" — a term that appears across ML, science, cooking, and skincare — and collected 459 sentences from Wikipedia using its API, spanning all four domains.

Using a pretrained BERT model, I extracted 768-dimensional embeddings for each occurrence of "layer" and applied t-SNE to reduce them to 2D for visualization. The resulting clusters reveal that BERT treats "layer" very differently depending on context: ML/AI forms a tight, isolated cluster (driven by specialized vocabulary like weights, attention, gradient), while science and skincare overlap due to their shared biological framing. Nearest-neighbor analysis confirmed that sentences about neural network layers consistently retrieve other ML sentences — even though the surface word is identical across all domains.

Skincare Recommendation System

Python HuggingFace ML

Built a full RAG pipeline deployed on Streamlit. Scraped product names, brands, and ingredients from INCIDecoder using requests + BeautifulSoup, storing results in a structured JSON file.

Each product is embedded into a 768-dimensional vector using HuggingFace's all-mpnet-base-v2 and indexed with FAISS for fast similarity search. When a user submits a query, the app retrieves the 5 most semantically relevant products and passes them as context to LLaMA 3.3 70B (via Groq) to generate a grounded recommendation. The FAISS index is cached on load so the app stays fast between interactions.

Arduino PCR Thermocycler

Arduino C / C++ Python Pandas Matplotlib

I designed and built a physical PCR thermocycler using Arduino. The hardware consists of a temperature sensor, a dual-relay module controlling independent heating and cooling elements, and a LCD that displays the current cycle, stage, and live temperature in real time.

The C++ firmware autonomously runs 30 PCR cycles, stepping through denaturation (95°C, 30s hold), annealing (55°C, 30s hold), and extension (72°C, 60s hold). It streams timestamped temperature readings over Serial every 0.5 seconds, which I captured as a CSV — over 10,000 data points across an ~83-minute run.

I then built a Python analysis pipeline (Pandas, Matplotlib) to validate the device's thermal precision. Deviation analysis confirmed every cycle held within ±1.5°C of each target, and z-score analysis showed no statistically significant outliers — a mean deviation under 0.1°C across all 30 cycles and three stages.
